üéØ How to Learn Effectively

This website uses research-proven methods to help you learn programming. Here's how to get the most from it:

1. Follow the PRIMM Method

Each topic follows these steps:

  • Predict - Look at code and guess what it does BEFORE running it
  • Run - See what actually happens and compare to your prediction
  • Investigate - Trace through the code step by step
  • Modify - Make small changes to existing code
  • Make - Create your own programs

This order is important! Don't skip to writing code before you understand how to read it.

2. Complete Activities in Order

Each topic has activities numbered 1, 2, 3, etc. Do them in order - each one builds on the previous.

3. Use the Hints

If you're stuck, use the hint buttons! There's no penalty for using hints, and they help you learn.

4. Don't Worry About Mistakes

Getting things wrong is part of learning. Professional programmers make mistakes all the time - that's why debugging skills matter!

üß© Activity Types

Prediction Quizzes

You'll see some code and need to predict what it will output. This builds your ability to "read" code like a programmer.

  • Take your time to trace through the code mentally
  • Think about what each line does in order
  • Don't guess randomly - make a reasoned prediction

Parsons Problems

Drag and drop code blocks into the correct order. This helps you learn code structure without worrying about typing.

  • Read the problem carefully - what should the output be?
  • Think about what needs to happen first, second, etc.
  • In Python, indentation matters! Make sure code inside loops is indented.
  • Some problems have "distractor" blocks that don't belong - watch out!

Code Tracing

Track what happens to variables as code runs. Fill in the trace table step by step.

  • Follow the code line by line, in order
  • Write down variable values AFTER each line runs
  • Use "-" or leave blank for variables that don't exist yet
  • Pay attention to loops - the same lines run multiple times!

Worked Examples

Step-by-step explanations of how to solve a problem. Study these before trying similar problems.

  • Click through each step to see the explanation
  • Pay attention to the "subgoal labels" - these describe WHAT each part does
  • Look at the "common mistakes" sections to avoid errors

üí° Tips for Success

  • Read code out loud - it helps you understand it better
  • Use paper - trace tables work great on paper too
  • Take breaks - if you're stuck, step away and come back
  • Ask for help - if you're really stuck, ask your teacher
  • Practice regularly - short, frequent practice beats long, rare sessions

üì± Technical Help

Progress Not Saving?

Your progress is saved in your browser's local storage. If you:

  • Use a different browser, your progress won't appear
  • Clear your browser data, your progress will be lost
  • Use private/incognito mode, progress won't be saved

Use the "Export Progress" button on the Progress page to save a backup!

Something Not Working?

Try these steps:

  1. Refresh the page
  2. Clear your browser cache
  3. Try a different browser (Chrome, Firefox, Edge all work well)
  4. Tell your teacher about the problem
