1 Characters

Text File Size in ASCII

Problem

A text file contains the sentence: "Hello World!" (12 characters including space and punctuation)

The file uses ASCII encoding (8 bits per character).

Calculate the file size in bits and bytes.

Solution

1

Identify the values

  • Number of characters = 12
  • Bits per character (ASCII) = 8 bits
2

Apply the formula

File size (bits) = Characters √ó Bits per character

File size (bits) = 12 √ó 8 = 96 bits

3

Convert to bytes

File size (bytes) = 96 √∑ 8 = 12 bytes

Answer

The file size is 96 bits or 12 bytes.

Notice: In ASCII, 1 character = 1 byte, so characters = bytes!

Comparing ASCII and Unicode File Sizes

Solution

1

Calculate ASCII file size

File size = 500 √ó 8 = 4,000 bits

In bytes: 4,000 √∑ 8 = 500 bytes

2

Calculate Unicode file size

File size = 500 √ó 16 = 8,000 bits

In bytes: 8,000 √∑ 8 = 1,000 bytes

3

Calculate the difference

Difference = 1,000 - 500 = 500 bytes

Or: 1,000 √∑ 500 = 2√ó larger

Answer

Unicode file is 500 bytes larger (double the size of ASCII).

Unicode uses 2√ó more bits per character, so files are 2√ó larger.

3 Characters

Finding ASCII Character Codes

Problem

In ASCII, the character 'A' has the code 65 and 'a' has the code 97.

What are the ASCII codes for 'G' and 'g'?

Solution

1

Find position of 'G' in alphabet

G is the 7th letter of the alphabet

2

Calculate uppercase 'G' code

A = 65, and G is 6 positions after A

G = 65 + 6 = 71

3

Calculate lowercase 'g' code

a = 97, and g is 6 positions after a

g = 97 + 6 = 103

Answer

'G' = 71, 'g' = 103

Pattern: Lowercase letters are always 32 more than uppercase (103 - 71 = 32)
