Knowledge Quiz

Test your understanding of data representation

Instructions

Answer all 15 questions to test your knowledge of characters, images, and sound representation. You'll see your results at the end.

Topics covered: Character sets (ASCII, Unicode), image file sizes (resolution, colour depth, metadata), sound files (sample rate, bit depth, duration)

Question 1

What is a character set?

Explanation: A character set defines which characters are available and assigns a unique binary code to each one.

Question 2

ASCII uses 8 bits per character. How many different characters can it represent?

Explanation: With n bits, you can represent 2^n characters. 2^8 = 256 characters.

Question 3

In ASCII, if 'A' = 65, what is the code for 'C'?

Explanation: ASCII characters are logically ordered. C is 2 positions after A, so 65 + 2 = 67.

Question 4

Which character set would you choose for a website displaying Japanese, Arabic, and emojis?

Explanation: ASCII only supports 256 characters (English letters and basic symbols). Unicode supports over 140,000 characters including all world languages and emojis.

Question 5

A text file contains 1,500 characters. Using Unicode (16-bit), what is the file size in bytes?

Explanation: 1,500 √ó 16 = 24,000 bits. 24,000 √∑ 8 = 3,000 bytes. (Or simply: 16 bits = 2 bytes per character, so 1,500 √ó 2 = 3,000)

Question 6

What is a pixel?

Explanation: A pixel (picture element) is the smallest single point in a digital image. Each pixel has a specific colour represented by a binary code.

Question 7

An image has a colour depth of 24 bits. How many different colours can each pixel display?

Explanation: Number of colours = 2^colour depth. 2^24 = 16,777,216 (approximately 16.7 million colours, known as "true colour").

Question 8

What is the file size of a 800 √ó 600 pixel image with 8-bit colour depth?

Explanation: Pixels = 800 √ó 600 = 480,000. Bits = 480,000 √ó 8 = 3,840,000. Bytes = 3,840,000 √∑ 8 = 480,000 bytes.

Question 9

Which of these is NOT typically stored in image metadata?

Explanation: Metadata is "data about data" - information about the image (dimensions, date, camera settings). The actual pixel colour values are the main image data, not metadata.

Question 10

What does "sample rate" measure in digital audio?

Explanation: Sample rate is measured in Hertz (Hz) and indicates how many times per second the sound wave is sampled. CD quality is 44,100 Hz (44.1 kHz).

Question 11

What is the standard sample rate for CD quality audio?

Explanation: CD quality audio uses 44,100 Hz (44.1 kHz) sample rate with 16-bit depth.

Question 12

What effect does increasing the bit depth have on a sound file?

Explanation: Bit depth determines how many different volume levels can be recorded. Higher bit depth = more precise audio with better dynamic range, but larger files.

Question 13

A mono audio file is 5 MB. What size would the same recording be in stereo?

Explanation: Stereo has 2 channels (left and right), each storing separate audio data. Stereo file size = mono √ó 2 = 10 MB.

Question 14

Calculate the file size of a 10-second mono sound at 44,100 Hz and 16-bit depth.

Explanation: Bits = 44,100 √ó 16 √ó 10 = 7,056,000 bits. Bytes = 7,056,000 √∑ 8 = 882,000 bytes.

Question 15

Which statement best describes the trade-off in data representation?

Explanation: The universal trade-off: more bits per unit (character, pixel, sample) = better quality/more options, but larger file sizes. This applies to text, images, and sound.
