Two Types of Errors

üî¥ Syntax Errors

What: Mistakes in how you write the code - breaking Python's grammar rules.

When detected: Before the program runs. Python won't even start.

Examples:

  • Missing colons after if/for/while
  • Incorrect indentation
  • Unclosed brackets or quotes
  • Misspelled keywords

Good news: Python tells you exactly where the error is!

üü° Logic Errors

What: Code runs without crashing, but produces wrong results.

When detected: Only when you test and check the output.

Examples:

  • Using wrong operator (+ instead of -)
  • Off-by-one errors in loops
  • Wrong comparison (> instead of >=)
  • Variables in wrong order

Challenge: No error message - you must find them yourself!

Key Vocabulary

syntax error

Breaking Python's grammar rules. The code cannot run at all.

logic error

Code runs but gives wrong results. Also called a "bug".

runtime error

Error that occurs while the program is running (e.g., dividing by zero).

debugging

The process of finding and fixing errors in code.

traceback

Python's error report showing where and what went wrong.

exception

A runtime error that stops the program (e.g., TypeError, ValueError).

Common Logic Errors

logic_errors.py 
# ‚ùå Off-by-one error (wrong range)
for i in range(1, 5):  # Goes 1,2,3,4 not 1,2,3,4,5!
    print(i)

# ‚ùå Wrong operator
average = total + count  # Should be total / count

# ‚ùå Wrong comparison
if score > 50:         # Misses exactly 50! Should be >= 50

# ‚ùå Variable not updated
total = 0
for num in numbers:
    total + num          # Should be: total = total + num

# ‚ùå Initialising inside loop
for num in numbers:
    total = 0            # Resets every iteration! Move before loop
    total = total + num

# ‚ùå Print inside loop (meant for after)
for num in numbers:
    total = total + num
    print(total)         # Prints every iteration! Move outside loop
