Predict: Protocols and Connections

Think about which protocols, connections, and addresses to use in different scenarios

How to Use This Activity

For each scenario, make your prediction before revealing the answer. Think about what you know about protocols, wired/wireless connections, and addressing.

Scenario 1: Gaming PC Setup

Alex is setting up a gaming PC in their bedroom. They want the fastest, most reliable internet connection for online gaming where lag could mean losing a match.

Question: Should Alex use a wired (Ethernet) or wireless (WiFi) connection? Why?

Answer: Wired (Ethernet)

Reasons:

  • Lower latency: Wired connections have less delay, crucial for competitive gaming
  • Consistent speed: No interference from walls or other devices
  • Higher bandwidth: Gigabit Ethernet provides faster speeds than most WiFi
  • Reliability: No risk of signal drops during intense gaming sessions

Since the gaming PC is stationary (doesn't need to move), the lack of mobility isn't a problem.

Scenario 2: Wireless Headphones

Sarah wants to connect wireless headphones to her phone to listen to music while walking around the house.

Question: Which wireless technology would be used - WiFi or Bluetooth?

Answer: Bluetooth

Reasons:

  • Designed for personal devices: Bluetooth is specifically made for connecting peripherals
  • Low power: Uses less battery than WiFi
  • Range is sufficient: 10m range is enough since headphones stay close to the phone
  • Simple pairing: Easy one-time setup between devices

WiFi would be overkill for this purpose and would drain battery faster.

Scenario 3: Secure Banking

Tom is about to log into his online banking to transfer money. He notices the website URL.

Question: Should the URL start with http:// or https://? Why does it matter?

Answer: https://

Reasons:

  • Encryption: HTTPS encrypts all data sent between browser and server
  • Protection: Login credentials and financial data cannot be intercepted
  • Verification: The padlock icon confirms the website's identity
  • Requirement: Banks MUST use HTTPS - if a banking site uses HTTP, it's likely fake!

Always look for the üîí padlock before entering sensitive information.

Scenario 4: Multi-Device Email Access

Emma checks her email on her laptop, phone, and tablet throughout the day. She wants to see the same inbox on all devices, and when she reads an email on one device, she wants it to show as read on all devices.

Question: Should Emma's email be configured to use POP3 or IMAP?

Answer: IMAP

Reasons:

  • Synchronisation: IMAP keeps emails on the server, syncing across all devices
  • Consistent state: Read/unread status, folders, and labels sync automatically
  • Multiple devices: IMAP is designed for accessing email from many devices
  • No duplicates: Unlike POP3, you won't have different emails on different devices

POP3 would download emails to one device and potentially delete them from the server, making multi-device access problematic.

Scenario 5: Website Deployment

A web developer has finished building a website on their computer. They now need to upload all the HTML, CSS, and image files to the web hosting server.

Question: Which protocol would they most likely use - HTTP, HTTPS, or FTP?

Answer: FTP (or SFTP)

Reasons:

  • File transfer: FTP is specifically designed for uploading and downloading files
  • Bulk transfers: Can upload many files and folders efficiently
  • Directory navigation: Can browse and manage files on the remote server
  • Resume capability: Can resume interrupted transfers

HTTP/HTTPS are for viewing web pages, not for uploading files to a server. Modern practice often uses SFTP (Secure FTP) for encrypted transfers.

Scenario 6: Identifying a Device

A network administrator needs to find a specific laptop that has been reported lost somewhere in the school building. The laptop is still connected to the school network.

Question: Would the admin use the laptop's IP address or MAC address to identify it? Why?

Answer: MAC Address

Reasons:

  • Permanent identifier: MAC address is fixed and unique to that specific laptop
  • Doesn't change: Unlike IP addresses which can be reassigned
  • Physical identification: Each NIC has a manufacturer-assigned MAC
  • Network tracking: Switches track which MAC addresses connect to which ports

The IP address might change if the laptop reconnects to a different part of the network, but the MAC address always stays the same.

Scenario 7: Smart Home Devices

A family is setting up several smart home devices: a video doorbell, smart light bulbs, a smart thermostat, and voice assistants. None of these devices have Ethernet ports.

Question: What type of connection will these devices use?

Answer: WiFi (Wireless)

Reasons:

  • No physical ports: Most IoT devices don't have Ethernet connections
  • Placement flexibility: Devices can be placed anywhere in range of WiFi
  • No cable installation: Practical for devices throughout the home
  • Always connected: WiFi provides constant connectivity for remote control

Some smart devices use Bluetooth for initial setup, then connect to WiFi for internet access.

Scenario 8: IPv4 Running Out

A news article mentions that IPv4 addresses have essentially run out globally. A student asks why we simply don't create more IPv4 addresses.

Question: Why can't we just create more IPv4 addresses? What's the solution?

Answer: IPv4 format limits the number possible

Explanation:

  • Fixed format: IPv4 uses 32 bits (4 numbers, 0-255 each)
  • Maximum addresses: 2^32 = approximately 4.3 billion addresses
  • More devices than addresses: With billions of phones, computers, IoT devices, we've run out
  • Can't change the format: Changing IPv4 would break the entire internet!

Solution: IPv6 uses 128 bits, providing 340 undecillion addresses - enough for every grain of sand on Earth to have its own address!

Scenario 9: Email Sending and Receiving

Maya sends an email from her Gmail to her friend who uses Outlook. The email successfully arrives.

Question: Which email protocols were used in this process?

Answer: SMTP (for sending) and IMAP or POP3 (for receiving)

The process:

  1. SMTP: Maya's email client sends the email to Gmail's mail server
  2. SMTP: Gmail's server forwards the email to Outlook's mail server
  3. IMAP/POP3: Maya's friend's email client downloads the email from Outlook's server

SMTP is always used for sending. The receiving protocol depends on how the recipient has configured their email client.

Scenario 10: Coffee Shop Connection

Jamie is working at a coffee shop using the free public WiFi. They need to check their bank balance and make a payment.

Question: What security concerns should Jamie have, and what should they check before logging in?

Answer: Public WiFi can be intercepted

Security concerns:

  • No encryption: Public WiFi often has weak or no encryption
  • Man-in-the-middle: Attackers on the same network could intercept traffic
  • Fake hotspots: The network might not even be the real coffee shop WiFi

What to check:

  • üîí Ensure the bank website uses HTTPS (encrypted connection)
  • Verify the URL is the correct bank address
  • Consider using a VPN for additional encryption
  • Avoid accessing sensitive accounts on public WiFi if possible

Scenario 11: Why Standards Matter

Imagine if every phone manufacturer used their own unique charging cable, every WiFi router used proprietary signals that only worked with that brand's devices, and every file format could only be opened by one specific program.

Question: What problems would this cause? How do standards help?

Answer: Standards enable interoperability

Problems without standards:

  • Vendor lock-in: Stuck with one manufacturer forever
  • Higher costs: No competition means higher prices
  • Incompatibility: Devices and software can't work together
  • Waste: Need different cables, adapters, software for everything
  • Limited choice: Can't choose the best product for your needs

How standards help:

  • Any USB device works with any USB port
  • Any WiFi device connects to any WiFi router
  • JPEG images open in any image viewer
  • Competition drives innovation and lowers prices

Scenario 12: Layered Troubleshooting

A user complains they can't load websites. An IT technician starts troubleshooting by checking if the network cable is plugged in.

Question: Using the concept of layers, why would the technician start with the physical connection rather than checking the web browser first?

Answer: Lower layers must work before higher layers can function

Layer-based troubleshooting:

  • Layer 1 (Network Access): Physical cable must be connected and working
  • Layer 2 (Internet): Device needs a valid IP address
  • Layer 3 (Transport): Data packets must be able to reach the server
  • Layer 4 (Application): Browser and website must work correctly

Why start at the bottom:

  • If the cable is unplugged (Layer 1), nothing else can work
  • Each layer depends on the layers below it
  • Start simple - check physical connections first
  • Work up through layers systematically to isolate the problem

This is why IT often asks "Is it plugged in?" first!
