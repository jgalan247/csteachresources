Why Network Security Matters

As more of our lives move online, computer systems become targets for attackers. Understanding threats helps us protect ourselves, our data, and the systems we use.

91% of cyber attacks start with phishing

People are the Weak Point: The most sophisticated security systems can be bypassed if attackers can trick people. Social engineering exploits human psychology rather than technical vulnerabilities.

1. Malware

Malware (malicious software) is any software designed to harm, exploit, or damage computer systems, networks, or users.

ü¶†

Virus

How it works: Attaches itself to legitimate programs or files. Spreads when the infected file is shared or executed.

Purpose: Damage files, corrupt systems, spread to other computers.

Example: A virus hidden in a downloaded game that corrupts your documents when run.
üêõ

Worm

How it works: Self-replicating malware that spreads across networks automatically without needing a host file.

Purpose: Spread rapidly, consume bandwidth, deliver other malware payloads.

Example: A worm that spreads via email, sending copies of itself to all contacts.
üê¥

Trojan

How it works: Disguises itself as legitimate software. Users willingly install it, not knowing it contains malicious code.

Purpose: Create backdoors for attackers, steal data, install other malware.

Example: A "free antivirus" that actually gives hackers access to your computer.
üí∞

Ransomware

How it works: Encrypts files on the victim's computer, making them inaccessible. Demands payment for the decryption key.

Purpose: Extort money from victims who need their files back.

Example: All your photos and documents are locked, with a message demanding ¬£500 in Bitcoin.
üëÅÔ∏è

Spyware

How it works: Secretly monitors user activity. May record keystrokes (keylogger), capture screenshots, or track browsing.

Purpose: Steal passwords, financial details, personal information, or monitor activity.

Example: Spyware that records every password you type and sends it to hackers.
üì¢

Adware

How it works: Displays unwanted advertisements, often through pop-ups or browser redirects.

Purpose: Generate advertising revenue for attackers, may also collect browsing data.

Example: Constant pop-up ads appear even when you're not browsing the web.

How Malware Spreads

  • Email attachments: Opening infected files attached to emails
  • Malicious downloads: Downloading software from untrusted sources
  • Infected websites: Visiting compromised websites (drive-by downloads)
  • USB drives: Plugging in infected removable media
  • Network vulnerabilities: Exploiting unpatched security holes

2. Social Engineering

Social engineering attacks manipulate people into revealing confidential information or performing actions that compromise security. These attacks exploit human psychology rather than technical vulnerabilities.

People are the Weak Point! Even the most secure computer systems can be compromised if an attacker can trick a person with access. Social engineering is extremely effective because it bypasses technical security measures.

Phishing

How it works: Attackers send fraudulent messages (usually emails) that appear to come from trusted sources (banks, companies, colleagues). The message tricks victims into:

  • Clicking malicious links
  • Entering login credentials on fake websites
  • Downloading malware attachments
  • Revealing sensitive information

Purpose: Steal login credentials, financial information, or install malware.

Example Phishing Email

From: security@amaz0n-support.com Subject: URGENT: Your account has been suspended!

Dear Customer,

We have detected unusual activity on your account. Your account has been temporarily suspended.

Click here to verify your identity: http://amaz0n-verify.suspicious-site.com/login

If you do not verify within 24 hours, your account will be permanently deleted.

Warning Signs:
  • Sender address uses "0" instead of "o" (amaz0n)
  • Creates urgency and fear
  • Link goes to suspicious domain, not amazon.com
  • Threatens account deletion

Other Social Engineering Techniques

Pretexting

Creating a fake scenario to gain trust. Example: Calling and pretending to be IT support to get login details.

Baiting

Leaving infected USB drives in public places, hoping someone will plug them into their computer.

Tailgating

Following an authorised person through a secure door without proper credentials.

Quid Pro Quo

Offering something (like free tech support) in exchange for information or access.

Protection: Always verify requests for sensitive information through official channels. Don't click links in unexpected emails - go directly to websites by typing the address.

3. Brute-Force Attacks

A brute-force attack attempts to crack passwords or encryption by systematically trying every possible combination until the correct one is found.

How It Works

  1. Attacker obtains a password hash or encrypted data
  2. Software automatically tries millions of password combinations
  3. Each guess is compared to the target until a match is found
  4. Simple passwords can be cracked in seconds; complex ones take years

Types of Brute-Force Attacks

Pure Brute-Force

Tries every possible combination of characters.

Tries: a, b, c... aa, ab, ac... aaa, aab...

Dictionary Attack

Uses a list of common passwords and words.

Tries: password, 123456, qwerty, letmein...

Hybrid Attack

Combines dictionary words with numbers/symbols.

Tries: password1, password123, P@ssword...

Purpose: Gain unauthorised access to accounts, systems, or encrypted data by discovering passwords or encryption keys.

A Denial of Service (DoS) attack aims to make a system, network, or website unavailable to legitimate users by overwhelming it with traffic or requests.

Types of DoS Attacks

DoS (Single Source)

Attack comes from a single computer or location.

  • Easier to block (block one IP)
  • Limited attack power
  • Less common today

DDoS (Distributed)

Attack comes from many computers simultaneously (often a botnet).

  • Very difficult to block
  • Massive attack power
  • Uses thousands of infected computers

What is a Botnet?

A botnet is a network of computers infected with malware and controlled by an attacker. The owners often don't know their computers are being used for attacks.

  • Computers in a botnet are called "zombies" or "bots"
  • Botnets can contain millions of infected devices
  • IoT devices (cameras, routers) are often targeted

Impact of DoS Attacks

  • Website downtime: Online shops, banks, services become inaccessible
  • Financial losses: Every minute of downtime costs money
  • Reputation damage: Customers lose trust in unreliable services
  • Distraction: May cover up other attacks happening simultaneously
Purpose: Disrupt services, cause financial damage, extort money (pay to stop the attack), or as a smokescreen for other malicious activities.

Methods of Data Interception

Packet Sniffing

Using software to capture network traffic. Effective on unencrypted connections.

WiFi Eavesdropping

Monitoring unsecured or poorly secured wireless networks.

Fake WiFi Hotspots

Creating a fake public WiFi (e.g., "Free_Coffee_Shop_WiFi") to capture all traffic.

ARP Spoofing

Redirecting network traffic through the attacker's computer.

What Data Can Be Stolen?

  • Login credentials: Usernames and passwords
  • Financial data: Credit card numbers, bank details
  • Personal information: Names, addresses, dates of birth
  • Session cookies: Allowing account takeover
  • Emails and messages: Private communications
Public WiFi Warning: Never enter sensitive information (banking, passwords) on public WiFi networks. Use HTTPS websites (look for the padlock üîí) and consider using a VPN.

6. SQL Injection

SQL Injection is an attack that inserts malicious SQL code into application queries, allowing attackers to manipulate databases.

How It Works

Websites often use forms (login boxes, search fields) that send user input to a database. If the website doesn't properly validate input, attackers can inject SQL commands.

Normal Login Process

User enters: Username: alice Password: secret123

Query becomes: SELECT * FROM users WHERE username='alice' AND password='secret123'

Result: Returns Alice's account if credentials are correct

SQL Injection Attack

Attacker enters: Username: ' OR '1'='1 Password: ' OR '1'='1

Query becomes: SELECT * FROM users WHERE username='' OR '1'='1' AND password='' OR '1'='1'

Result: '1'='1' is always TRUE - returns ALL users! Attacker gains access!

What Attackers Can Do

  • Bypass authentication: Log in without valid credentials
  • Access data: Retrieve sensitive information from the database
  • Modify data: Change prices, grades, account balances
  • Delete data: Drop tables, destroy records
  • Execute commands: Sometimes run system commands on the server

Real-World Impact

SQL injection has been used in major data breaches affecting millions of users. It consistently ranks in the OWASP Top 10 most critical web security risks.

Example attacks:

  • 2017 Equifax breach - 147 million people affected
  • TalkTalk breach - 157,000 customers' data stolen
  • Sony Pictures hack - internal data leaked
Purpose: Gain unauthorised access to databases, steal sensitive data, modify or delete records, or take control of the server.
