Worked Examples: Utility Software

Study model answers to exam-style questions

Study these worked examples carefully. Pay attention to the structure of the answers and the key terminology used. Then try the practice questions at the end.

Worked Example 1 [4 marks]

Question

Explain the purpose of encryption software and give two reasons why it is needed.

Model Answer

Purpose: Encryption software scrambles data into an unreadable format (ciphertext) that can only be converted back to readable text using a decryption key. [1]

Reason 1: It protects sensitive data if a device is lost or stolen. Even if someone gains physical access to the storage device, they cannot read the encrypted files without the key. [1¬Ω]

Reason 2: It ensures data transmitted over networks remains confidential. If data is intercepted during transmission, it appears as meaningless characters without the decryption key. [1¬Ω]

Why This Answer Works

  • Clear definition: Explains what encryption does (scrambles to ciphertext, needs key to decrypt)
  • Two distinct reasons: Covers both storage security and transmission security
  • Technical terminology: Uses terms like "ciphertext" and "decryption key"
  • Practical examples: References lost devices and intercepted data
Worked Example 2 [4 marks]

Question

Describe how defragmentation software improves the performance of a hard disk drive (HDD).

Model Answer

Over time, files become fragmented - stored in non-contiguous (scattered) locations across the disk. [1]

This happens because when files are deleted, gaps are created, and new files are saved in whatever space is available, often split across multiple locations. [1]

Defragmentation software reorganises the data so that files are stored in contiguous (adjacent) blocks. [1]

This improves performance because the HDD's read/write head doesn't need to move to multiple locations to access a single file, reducing seek time and making file access faster. [1]

Why This Answer Works

  • Explains the problem: Describes how fragmentation occurs
  • Explains the solution: Describes what defragmentation does
  • Links to hardware: Mentions read/write head movement
  • Key terms: Uses "contiguous", "fragmented", "seek time"
Worked Example 3 [6 marks]

Question

Compare lossy and lossless compression. Include in your answer when each type would be appropriate to use.

Model Answer

Lossy compression permanently removes some data from the file to achieve high compression ratios. The original file cannot be perfectly restored. [1]

Lossless compression reduces file size without losing any data. The original file can be perfectly reconstructed from the compressed version. [1]

Lossy compression achieves much higher compression ratios than lossless (often 90%+ reduction) because it discards data deemed less important. [1]

Lossy is appropriate for media files (images, audio, video) where small quality loss is acceptable and smaller file sizes are important for streaming or web use. Examples: JPEG, MP3, MP4. [1¬Ω]

Lossless is appropriate for text documents, source code, and executable files where every byte must be preserved exactly. Even one changed bit could corrupt a program or change the meaning of a document. Examples: ZIP, PNG. [1¬Ω]

Why This Answer Works

  • Clear comparison: Explains both types with their key difference
  • Compression ratio mentioned: Notes lossy achieves higher compression
  • Appropriate use cases: Gives specific examples for each type
  • File format examples: Names actual formats (JPEG, MP3, ZIP, PNG)
Worked Example 4 [3 marks]

Question

Explain why utility software is needed in addition to the operating system.

Model Answer

The operating system handles core functions such as memory management, process scheduling, file system management, and hardware communication. [1]

Utility software provides additional housekeeping tasks that the operating system may not include or may only offer in basic form. [1]

These tasks include encryption for security, defragmentation for HDD performance optimisation, and compression for reducing file sizes. Third-party utilities often provide more features than built-in OS tools. [1]

Why This Answer Works

  • Acknowledges OS role: Shows understanding of what the OS does
  • Key phrase: Uses "housekeeping tasks"
  • Specific examples: Lists actual utility types
  • Third-party note: Mentions additional software options

Practice Question 1 [2 marks]

State two purposes of data compression software.

Model Answer

  1. Reduce file size to save storage space - allows more files to be stored on a disk [1]
  2. Reduce transmission time when sending files - smaller files transfer faster over networks [1]

Other acceptable answers: reduce bandwidth usage, allow files to fit within email attachment limits, create archives combining multiple files

Practice Question 2 [3 marks]

Explain why you should not defragment a solid-state drive (SSD).

Model Answer

SSDs have no moving parts - they can access any storage location equally fast, so fragmentation doesn't cause performance issues. [1]

SSDs have a limited number of write cycles. Defragmentation involves many unnecessary write operations. [1]

These unnecessary writes reduce the lifespan of the SSD without providing any performance benefit. [1]

Practice Question 3 [4 marks]

A hospital stores patient medical records on computer systems. Explain why encryption is essential for this data and describe how it protects patient information.

Model Answer

Medical records contain sensitive personal data that must be kept confidential under laws like GDPR. [1]

Encryption scrambles the data into unreadable ciphertext that cannot be understood without the decryption key. [1]

If a computer is hacked or a storage device is stolen, the encrypted data remains protected - the attacker cannot read patient information. [1]

When data is transmitted between hospital systems, encryption ensures it cannot be intercepted and read by unauthorised parties. [1]
