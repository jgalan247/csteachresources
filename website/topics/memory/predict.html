Predict: Memory Scenarios

Think about what happens in different memory situations before revealing the answers.

How to use this activity:

  1. Read each scenario carefully
  2. Think about what will happen based on your understanding
  3. Click "Reveal Answer" to check your prediction
  4. Read the explanation to deepen your understanding

Scenario 1: Power Cut

You're working on an essay in Word. You haven't saved it yet. Suddenly, there's a power cut and your computer turns off.

What happens to your unsaved essay? Why?

Answer:

Your essay is lost.

The essay was stored in RAM, which is volatile. This means RAM loses all its contents when the power is switched off. Until you save a file, it only exists in RAM. Once saved, it's copied to secondary storage (hard drive/SSD) which is non-volatile.

Key concept: RAM is volatile - always save your work!

Scenario 2: Computer Boot

You press the power button on your computer. The RAM is empty (it was off). The hard drive contains Windows, but the CPU can't read from the hard drive directly at startup.

How does the computer know what to do when you first turn it on?

Answer:

The BIOS stored in ROM tells it what to do.

ROM is non-volatile - it keeps its contents even when power is off. The BIOS (Basic Input/Output System) is stored in ROM and contains the instructions needed to:

  1. Test the hardware (POST - Power On Self Test)
  2. Find the operating system on the hard drive
  3. Load the OS into RAM
Key concept: ROM stores permanent boot instructions (BIOS)

Scenario 3: Too Many Programs

Your computer has 8GB of RAM. You open:

  • Chrome with 20 tabs (using 4GB)
  • Photoshop with a large image (using 3GB)
  • Spotify (using 0.5GB)
  • A video game (needs 2GB)

That's 9.5GB total, but you only have 8GB RAM!

What happens? Does the computer crash?

Answer:

The computer uses virtual memory.

When RAM is full, the operating system uses virtual memory:

  1. Identifies least recently used data in RAM
  2. Moves it to a "page file" on the hard drive
  3. Frees up RAM for the new program
  4. If old data is needed, swaps it back

The computer won't crash, but it will slow down significantly because the hard drive is much slower than RAM.

Key concept: Virtual memory prevents crashes but causes slowdowns

Scenario 4: Disk Thrashing

A computer with only 4GB RAM is running many programs. The user notices:

  • The hard drive light is constantly on
  • A whirring sound from the hard drive
  • Everything is extremely slow
  • Even moving the mouse is laggy

What is causing this problem?

Answer:

Disk thrashing due to excessive virtual memory usage.

When RAM is severely overloaded, the system constantly swaps data between RAM and the hard drive. This is called disk thrashing:

  • Program A needs data ‚Üí swapped into RAM
  • Program B needs its data ‚Üí A's data swapped out
  • Program A needs data again ‚Üí B's data swapped out
  • This cycle repeats constantly

Solution: Close programs, or add more RAM to the computer.

Key concept: More RAM = less virtual memory needed = faster computer

Scenario 5: Cache Benefits

A CPU needs to access the same piece of data 1000 times during a calculation.

  • Reading from RAM takes 100 nanoseconds
  • Reading from cache takes 1 nanosecond

How much time does cache save if the data is found in cache 99% of the time?

Answer:

Cache saves approximately 98,000 nanoseconds (98 microseconds).

Without cache (all from RAM):

1000 √ó 100ns = 100,000 nanoseconds

With cache (99% hit rate):

990 √ó 1ns + 10 √ó 100ns = 990 + 1000 = 1,990 nanoseconds

Time saved: 100,000 - 1,990 = 98,010 nanoseconds

Cache is roughly 100√ó faster than RAM, making it crucial for performance. Even though cache is tiny compared to RAM, a high "hit rate" makes a huge difference.

Key concept: Cache dramatically speeds up repeated data access

Scenario 6: ROM Update

A manufacturer discovers a bug in the BIOS that could cause problems. They release a "BIOS update".

Wait - if ROM is "Read Only", how can it be updated?

Answer:

Modern computers use EEPROM or Flash ROM.

Traditional ROM truly couldn't be changed. However, modern "ROM" is usually:

  • EEPROM (Electrically Erasable Programmable ROM) - can be rewritten with special software
  • Flash ROM - similar to EEPROM, easier to update

We still call it "ROM" because:

  • It's non-volatile (keeps data without power)
  • It's not meant for regular writing
  • Updates are rare (unlike RAM which changes constantly)
Key concept: "ROM" in modern computers can technically be updated, but rarely is

Scenario 7: SSD vs HDD Virtual Memory

Two identical computers both run out of RAM and start using virtual memory. The only difference:

  • Computer A has an HDD (spinning hard drive)
  • Computer B has an SSD (solid state drive)

Which computer will perform better? Why?

Answer:

Computer B (SSD) will perform much better.

Virtual memory requires reading and writing to secondary storage. SSDs are much faster than HDDs:

Storage Type Typical Speed
HDD 100-200 MB/s
SSD (SATA) 500-600 MB/s
SSD (NVMe) 3,000-7,000 MB/s

This is why upgrading to an SSD makes such a noticeable difference, especially on computers with limited RAM.

Key concept: Faster storage = faster virtual memory = better performance when RAM is full

Scenario 8: Memory Hierarchy

The CPU needs to read a piece of data. It checks in this order:

  1. L1 Cache - NOT FOUND
  2. L2 Cache - NOT FOUND
  3. L3 Cache - NOT FOUND
  4. RAM - NOT FOUND

Where will the data come from next? What is this situation called?

Answer:

The data will come from virtual memory (hard drive). This is a "page fault".

When data isn't in RAM, it must be in the page file on secondary storage. The OS must:

  1. Find the data in the page file
  2. Load it into RAM (possibly swapping something else out)
  3. Then the CPU can access it

This is called a page fault - not an error, but it causes significant delay.

Key concept: Page faults are slow - data must be fetched from secondary storage

Scenario 9: Gaming Computer

A gamer is building a new PC. They have a choice:

  • Option A: 8GB RAM + 16GB virtual memory
  • Option B: 16GB RAM + 8GB virtual memory

Both have 24GB total "memory" available.

Which option would give better gaming performance? Why?

Answer:

Option B (16GB RAM) would be much better.

Virtual memory is NOT equivalent to RAM:

  • RAM is approximately 100,000√ó faster than an HDD
  • Even with an SSD, RAM is still 100√ó faster
  • Games need fast, constant access to data
  • Virtual memory causes stuttering and lag

16GB of real RAM means the game can run entirely from fast memory, avoiding slow virtual memory access.

Key concept: RAM and virtual memory are NOT equivalent - always prefer more RAM

Scenario 10: Smartphone Memory

Your smartphone has 6GB RAM but no virtual memory (most phones don't use it). You try to open too many apps.

What happens when the phone runs out of RAM?

Answer:

The phone closes background apps automatically.

Unlike computers, most smartphones don't use virtual memory because:

  • Flash storage has limited write cycles
  • Constant writing would wear out the storage
  • Battery life would suffer

Instead, the OS uses "app killing":

  • Monitors RAM usage
  • When full, closes least-recently-used apps
  • Apps reload when you switch back to them
Key concept: Different devices manage memory differently based on their hardware limitations
