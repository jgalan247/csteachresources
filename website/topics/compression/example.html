1 Choosing Type

Compressing a Software Installation File

Scenario

A software company needs to compress their 500 MB installation package for download from their website.

Which type of compression should they use, and why?

Solution

1

Identify the file type

Software installation files contain:

  • Executable code (.exe files)
  • Configuration files
  • Libraries and resources

These are program files - every bit matters!

2

Consider the consequences of data loss

If even one bit is changed or lost:

  • The software might crash
  • Security could be compromised
  • Features might not work
  • The entire installation could fail
3

Apply the decision rule

Rule: If the file must be exactly the same after decompression, use lossless.

Software files MUST be identical ‚Üí Use Lossless

Answer

Lossless compression (e.g., ZIP format)

Reason: Software files must be bit-perfect to function correctly. Any data loss would corrupt the installation.

This is why software downloads typically come as .zip, .7z, or self-extracting archives.

Storing Holiday Photos

Scenario

You've taken 200 photos on holiday, each 25 MB in RAW format. Your phone has 5 GB of storage space remaining.

What compression approach would allow you to keep all photos?

Solution

1

Calculate total uncompressed size

Total size = 200 photos √ó 25 MB = 5,000 MB (5 GB)

Available space = 5 GB

Problem: Files exactly fill the space with no room left!

2

Consider lossless compression

Lossless typically reduces photos by 20-40%

Best case: 5,000 √ó 0.6 = 3,000 MB (3 GB)

This might fit, but photos don't compress well with lossless.

3

Consider lossy compression (JPEG)

JPEG typically achieves 80-95% compression

At 90% compression: 5,000 √ó 0.1 = 500 MB

This easily fits with plenty of room to spare!

4

Assess acceptability of quality loss

Holiday photos are for:

  • Viewing on screens
  • Sharing on social media
  • Printing at normal sizes

Slight quality reduction is not noticeable for these uses.

Answer

Lossy compression (JPEG)

Result: 200 photos at ~2.5 MB each = 500 MB total

Reason: Photos compress very well with JPEG, and the quality difference is imperceptible for normal viewing and sharing.

Professional photographers might keep RAW files for editing, but JPEGs are fine for personal use.

8 Effects

Understanding Generation Loss

Scenario

A student edits a JPEG photo and saves it. Then edits it again and saves it. They repeat this 10 times.

What happens to the image quality, and why?

Solution

1

Understand the problem

JPEG is a lossy format. Each time you save:

  • The image is recompressed
  • More data is discarded
  • Quality degrades further

This is called generation loss.

2

Visualize the degradation

Each save cycle:

  • Save 1: Small quality loss (barely visible)
  • Save 2-3: Quality loss accumulates
  • Save 5-6: Visible artifacts appear
  • Save 10: Significant degradation, blocky areas
3

The solution

To avoid generation loss:

  • Edit in a lossless format (PNG, TIFF)
  • Only save as JPEG once, at the very end
  • Keep the original file unedited

Answer

Effect: After 10 saves, the image quality will be significantly degraded with visible compression artifacts (blocky areas, colour banding, blurring).

Why: JPEG compression discards data each time. These losses accumulate with each save cycle.

Prevention: Always edit in a lossless format and export to JPEG only once as the final step.

This is why professional photographers work with RAW files, not JPEGs!
