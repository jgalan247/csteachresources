1. The Need for Compression

Why Do We Need Compression?

Digital files can be very large. Without compression, many everyday tasks would be impractical or impossible:

üìß

Email Attachments

Most email services limit attachments to 25MB. An uncompressed photo could be 20MB+!

üéµ

Music Streaming

An uncompressed 3-minute song is ~30MB. With compression (MP3), it's only ~3MB.

üì±

Limited Storage

Phones have limited space. Compression lets you store thousands of photos instead of hundreds.

üåê

Web Pages

Compressed images load faster, saving bandwidth and improving user experience.

üé¨

Video Streaming

A 2-hour 4K movie uncompressed would be ~1TB! Compression makes streaming possible.

üíæ

Backups

Compressing backups saves storage space and reduces backup time.

What is Compression?

Compression is the process of reducing the file size by encoding data more efficiently.

Key Idea: Compression finds patterns and redundancy in data and represents them more efficiently, using fewer bits to store the same (or similar) information.

2. Types of Compression

Two Main Types

Lossless Compression

"Shrink it, but keep everything"

The original data can be perfectly reconstructed when decompressed.

No data is lost.

Lossy Compression

"Shrink it more, lose some detail"

Some data is permanently removed to achieve smaller file sizes.

Some data is lost.

3. Lossless Compression

How Lossless Works (Simplified)

Lossless compression finds patterns and redundancy in data and encodes them more efficiently.

Simple Example: Run-Length Encoding

Original: AAAAABBBCCCCCCCC (16 characters)

Compressed: 5A3B8C (6 characters)

The pattern "5 A's, 3 B's, 8 C's" uses fewer characters but contains the same information!

Key Property: When you decompress a lossless file, you get back exactly the original data, bit for bit.

When to Use Lossless

üìù

Text Documents

Every character matters - you can't lose any text!

üíª

Program Files / Software

A single bit change could break the program

üìä

Spreadsheets / Databases

Financial data, records - accuracy is critical

üé®

Graphics with Sharp Edges

Logos, diagrams, screenshots - PNG is ideal

üè•

Medical Images

Doctors need every detail for accurate diagnosis

Advantages & Disadvantages of Lossless

Advantages

  • No quality loss - original perfectly preserved
  • Can compress and decompress repeatedly
  • Essential for text, code, and critical data
  • Reversible - always get original back

Disadvantages

  • Lower compression ratio (files still larger)
  • Not as effective for photos/video
  • Files may still be too large for some uses

4. Lossy Compression

How Lossy Works (Simplified)

Lossy compression achieves smaller files by permanently removing data that humans are less likely to notice.

Examples of What Gets Removed

  • Images: Subtle colour variations, fine details
  • Audio: Frequencies humans can't hear well (very high/low)
  • Video: Details between similar frames
Important: Once data is removed by lossy compression, it's gone forever. You cannot get it back by "decompressing" - there's nothing to decompress!

Advantages & Disadvantages of Lossy

Advantages

  • Much smaller file sizes (often 90%+ reduction)
  • Makes streaming media practical
  • Faster uploads/downloads
  • Saves significant storage space
  • Quality loss often imperceptible

Disadvantages

  • Quality loss is permanent and irreversible
  • Re-compressing degrades quality further
  • Not suitable for text, code, or critical data
  • Visible/audible artifacts at high compression

Compression Artifacts

When lossy compression is too aggressive, you may notice artifacts - visible or audible flaws.

JPEG Artifacts (Images)

  • Blockiness: Visible squares, especially in gradients
  • Colour banding: Smooth gradients become stepped
  • Blurring: Loss of fine detail
  • Ringing: Halos around sharp edges

MP3 Artifacts (Audio)

  • Pre-echo: Ghost sounds before loud notes
  • Swirling: Metallic sound on cymbals
  • Muffled highs: High frequencies sound dull
